CI/CD Automation Documentation

This project implements a real-world CI/CD pipeline using Jenkins, Ansible, GitHub, and Apache. Any code pushed to GitHub is automatically deployed to the target server without manual intervention.

System Architecture

Developer â†’ GitHub â†’ Jenkins (Ansible) â†’ Target Server (Apache)

Jenkins acts as the automation engine while Ansible performs configuration and deployment on the target node.

Components Used

  • Jenkins: CI/CD automation server
  • Ansible: Agentless configuration management
  • GitHub: Source code management
  • Apache: Web server on target node

Ansible Inventory

[web]
192.168.107.139 ansible_user=shubham

Inventory defines which hosts Ansible manages. Without inventory, Ansible defaults to localhost.

Ansible Playbook

- hosts: web
  become: yes
  tasks:
    - copy:
        src: index.html
        dest: /var/www/html/index.html

    - service:
        name: apache2
        state: restarted

This playbook deploys website code and restarts Apache automatically.

Jenkins Job

ansible-playbook \
-i /var/lib/jenkins/Ansible/inventory \
webserver_update.yml

Jenkins pulls code from GitHub and executes Ansible for deployment.

Problems Faced & Solutions

  • No hosts matched: Inventory not passed â†’ fixed with -i inventory
  • sudo password error: Enabled passwordless sudo
  • SSH failed: Implemented SSH key authentication
  • Permission denied: Fixed ownership for Jenkins user

DevOps Cheat Sheet

ansible web -m ping -i inventory

ansible-playbook playbook.yml --syntax-check

Best Practices

  • Always use SSH keys
  • Never hardcode passwords
  • Use roles for scalability
  • Test playbooks before production

Conclusion

This project demonstrates a production-ready DevOps CI/CD pipeline. It follows industry standards and solves real automation challenges.
