‚ùå Common Problems & Solutions

1. No hosts matched

Problem:

skipping: no hosts matched

Solution:

  • Create inventory file
  • Use correct host group
  • Pass inventory with -i
ansible-playbook -i inventory playbook.yml

2. Permission denied (publickey,password)

Problem: SSH authentication fails

Solution:

  • Generate SSH key for Jenkins
  • Copy key to target server
ssh-keygen
ssh-copy-id user@server

3. sudo password required

Problem: Ansible fails during sudo tasks

Solution:

  • Disable fact gathering OR
  • Enable passwordless sudo
shubham ALL=(ALL) NOPASSWD:ALL
